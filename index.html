FraminghamPACE

Research Patented · Licensed

Aging unfolds across organ systems, leaving its mark in your bloodwork over time. Plotting nearly 40 years of longitudinal biomarker data from the Framingham Heart Study (right) you see an immense amount of variability, but each individual carries a coherent signal across biomarkers.

To distill these data into a single latent aging phenotype, I designed a hierarchical mixed effects model producing robust, per-person estimates of the pace of aging. By training a model to predict pace of aging from epigenomic data, an individual's score can be measured from a single blood draw.

The resulting epigenetic clock, FraminghamPACE, is now patented with me as co-lead inventor, licensed to TruDiagnostic, and the subject of a first-author manuscript in preparation.

QueryGaP

AI Tool Web App

dbGaP is one of the largest repositories of genotype-phenotype data in the world. The online site serves data documentation for 3,000 studies, including many that I work with daily. Tired of manually surfing their documentation online, I built QueryGaP.

To build it, I scraped the full dbGaP catalog (3,000 studies, 450,000 variables) into a PostgreSQL database with hybrid search (tsvector for keyword matching, pgvector for semantic similarity), then built a ReAct-style AI agent that queries the database and returns relevant results conversationally from a natural language prompt. Deployed publicly.

Latent Factor Allocation

Research Bayesian

For my MPhil dissertation at Cambridge, I worked on Latent Factor Allocation (LFA), a probabilistic topic model designed to discover genetic variants that underpin propensity to multiple disease types.

Exact inference methods exist (e.g. MCMC), but they don't scale. I derived a mean-field variational inference algorithm that makes the model tractable on datasets the size of the UK Biobank. I implemented this algorithm and a partially collapsed Gibbs sampler from scratch in NumPy and SciPy to compare these methods.

I've continued on this project as a part-time Visiting Researcher at Cambridge.

OpenCode Protect

Security Open Source

AI coding agents have full filesystem access by default. Existing protections are largely at the prompt-level, easily bypassed by injection attacks.

Often working with sensitive data, I also want to ensure that I can use AI coding agents without risking agents reading sensitive files.

OpenCode Protect solves these problems at the OS level. Commands run as a restricted Unix user without permission to access protected files, providing kernel-level security for sensitive files.

Web App Fun
